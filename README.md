# Remy
Ai 로봇 개발자 교육 2025.10.

> 시각장애인이 주방에서 **칼을 안전하게 잡고**, 선택한 도구까지 **손을 유도**할 수 있도록  
> **객체 인식 + LED(8방향/중앙/전체) + 오디오 알림**을 결합한 온디바이스 보조 시스템입니다.

---

## 📌 프로젝트 개요

- **문제**: 주방 조리 과정에서 시각장애인의 **베임(칼) 위험**이 높고, 안전한 파지/이동을 돕는 실시간 보조가 필요
- **해결**: 카메라 입력을 통해 **칼날/손잡이 분리 인식 + 손(검지 끝) 기반 위치 추정**을 수행하고  
  **LED 방향 안내 + 위험 경고 + 도달 확인**을 제공합니다.
- **플랫폼**: Raspberry Pi 5 (온디바이스 추론)

---

## 🧑‍💻 내 역할 (유진영)

- PM 역할 수행 (일정/통합 관점)
- **메인 기능 구현**
- **통합 테스트 및 디버깅**
- 임베디드 성능 문제 해결(온디바이스 실시간 동작 확보)

---

## ✨ 핵심 기능

### 1) 위험 객체 감지 & 안전 파지 안내
- **칼날/손잡이 분리 인식**
- 손이 위험 구역(칼날)에 접근하면 즉시 경고

### 2) 목표 도구 선택 & 방향 안내 (LED)
- 버튼으로 목표 도구(예: 국자/포크 등) 선택
- **검지 끝 랜드마크** 기준으로 목표까지의 상대 위치를 계산해 **8방향 LED**로 유도

### 3) 위험/도달 알림 (오디오 + LED)
- 위험 접근 시: **중앙 LED + 경고음**
- 목표 도달 시: **전체(9개) LED + 확인음**
- 이동 중 위험(칼날)이 경로에 있어도 **안내는 지속하면서 경고는 유지**

---

## 🧠 시스템 동작 흐름

1. 카메라 입력(UVC) 수신  
2. 객체 탐지(칼/도구 등) 및 손(검지 끝) 위치 추정  
3. bbox 기반 상대 위치 계산  
4. GPIO로 LED 제어 + 오디오 출력  
5. 버튼 입력으로 목표 변경

## ⚡ 성능 최적화 / 트러블슈팅

### ✅ 온디바이스 추론 지연 문제
- 초기: 라즈베리파이에서 추론 시 지연/프레임 저하로 실시간 사용 어려움
- 개선: **NCNN 적용**으로 임베디드 환경에 맞게 변환/최적화하여 **실시간 동작 확보**

### ✅ 오디오 출력으로 인한 프레임 드랍
- 문제: `sounddevice.sd.play()`를 프레임마다 호출하면서 오버헤드 발생 → 웹캠 멈춤/프레임 드랍
- 개선: **스트림 + 콜백 방식**으로 변경하여 안정화


## 🧰 기술 스택

- **Computer Vision**: YOLO 계열 객체 탐지, bbox 기반 상대 위치 계산
- **On-device**: Raspberry Pi 5, NCNN 최적화
- **Interaction**: GPIO(LED/버튼), 오디오 출력(스트림 기반)
